# Бот сбора обратной связи — MVP «Учёт ТМЦ»

Telegram-бот: его добавляют в групповой чат тестировщиков, он молча логирует все
сообщения (текст + скриншоты) в SQLite, а скриншоты скачивает в `screenshots/`.
Никаких анкет — люди пишут как привыкли. Структуру (экран, баг/идея, критичность)
по контексту восстанавливает Claude при разборе.

## Что собирает

Каждое сообщение группы/лички: кто (Telegram id/username/имя), когда (UTC),
текст, ответ-на, и скриншот (скачивается в файл). В личке бот тоже принимает
свободные сообщения.

> Чтобы бот видел ВСЕ сообщения группы, у него отключён Group Privacy в
> [@BotFather](https://t.me/BotFather) (`/setprivacy` → Disable), после чего он
> заново добавлен в чат.

## Команды (только для админов из `ADMIN_IDS`)

| Команда       | Что делает                                                     |
|---------------|---------------------------------------------------------------|
| `/export`     | новое **с прошлой выгрузки** (Markdown + JSON), двигает метку  |
| `/export_all` | вся переписка целиком; метку не трогает                        |
| `/stats`      | сколько сообщений/скринов, по каким чатам                      |
| `/whereami`   | id текущего чата                                               |
| `/help`       | краткая справка                                                |

**Раз в день** (по умолчанию 06:00 UTC = 09:00 МСК) бот сам присылает
администраторам инкрементальную выгрузку в личку. Час меняется через
`DAILY_EXPORT_HOUR_UTC`.

«Метка выгрузки» хранится в самой БД (таблица `export_state`): и `/export`, и
авто-выгрузка отдают только то, что появилось после предыдущей выгрузки, и
сдвигают метку. `/export_all` — независимая ручка на полный дамп.

## Быстрый старт (локально, для проверки)

```bash
cd feedback-bot
python -m venv .venv
.venv/bin/pip install -r requirements.txt      # Windows: .venv\Scripts\pip
cp .env.example .env    # впиши BOT_TOKEN и ADMIN_IDS
python bot.py
```

Токен — у [@BotFather](https://t.me/BotFather) (`/newbot`). Свой Telegram id — у
[@userinfobot](https://t.me/userinfobot); впиши в `ADMIN_IDS`, иначе `/export` и
авто-выгрузка недоступны.

## Деплой на сервер (автообновление через git)

На сервере `/root/feedback-bot` — это git-clone этого репозитория, запущенный как
systemd-служба `feedback-bot`. Раз в минуту `autopull.sh` (в cron) делает
`git pull` и при изменениях переустанавливает зависимости и перезапускает бота.
Правки прилетают автоматически: `git push` в `origin/main` → сервер подтянет за
~1–2 минуты. `.env`, `feedback.db` и `screenshots/` в `.gitignore` — на сервере
локальные.

## Как отдавать ОС в Claude

1. Возьми выгрузку (`/export`, `/export_all` или ежедневную авто-выгрузку).
2. Бот пришлёт `feedback-*.md` и `feedback-*.json`.
3. Кинь файл в чат с Claude Code: «разбери и собери бэклог / приоритизируй».

> Скриншоты в `.md`/`.json` указаны именами файлов из `screenshots/` на сервере.
> Чтобы Claude увидел сами картинки — пришли нужные скрины в чат отдельно или
> перетащи папку `screenshots/` на ПК через MobaXterm SFTP.
